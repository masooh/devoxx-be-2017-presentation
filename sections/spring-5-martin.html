Spring 5

- baseline Java 8 - support JUnit 5
- functional (Instance) Supplier interface -> keine Reflection - Foo::new (Methodenreferenz) -> kein Class.newInstance() (auch Reflection) - für kleine Microservices gedacht, kombinierbar mit Scanning
- Kotlin extensions in Spring enthalten - Vorteil Kotlin kennt return type von Generic
- neue Logging Bridge spring-jcl - kann mit allen gängigen umgehen - kein andere bridge oder exclude mehr notwendig
- Component Scan konnt bislang nur durch Verringerung des Suchraumes verbessert werden - jetzt wird zu Build Zeit Index aufgebaut und bei runtime verwendet
- Spring 5 Code ist nun per default non-null, sonst @Nullable annotiert - Treiber Kotlin's non-null variables wie von Leonard berichtet
- data binding mit immutable classes - properties als Parameter-Namen (entweder -parameter oder Annotation)
- Spring Boot 2 -> basiert auf Spring 5, neuer webflux Starter - Router Bean -> @Controller von Annotations befreien - hier Kotlin, sehr kurz - Talk: Umstellung Kotlin, Rest Anpassungen (leider vermischt)
